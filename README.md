# 📄 Система Автоматической Генерации Ответов (AI Responder)

Микросервисная ML-система для обработки входящих обращений, поиска нормативно-правовых актов (RAG) и автоматической генерации официальных ответов.

---

## ✨ Ключевые возможности

* **Обработка сканов:** Извлечение текста из прикрепленных PDF и изображений (OCR).
* **Умный анализ:** Автоматическое определение тематики жалобы и извлечение данных заявителя (ФИО, адрес).
* **Юридическая точность (RAG):** Семантический поиск по базе регламентов и законов для аргументации ответа.
* **Генерация текста:** Создание черновика ответа с помощью локальной языковой модели (LLM) без утечки персональных данных.
* **Оформление по ГОСТ:** Автоматическая сборка итогового документа (PDF/DOCX) для проверки человеком (Human-in-the-loop).

---

## 🏗 Архитектура пайплайна

Система работает асинхронно через шину данных **RabbitMQ** и состоит из 5 независимых модулей:

1. **API Gateway & OCR:** Принимает запросы по REST API, валидирует данные (Pydantic) и распознает сканы. Ставит задачу в очередь.
2. **Модуль NLP:** Забирает задачу из очереди. Определяет тематику (RuBERT) и находит ФИО (Natasha). Сохраняет метаданные в БД.
3. **Модуль RAG:** Векторизует текст жалобы (BGE-m3) и ищет релевантные статьи законов в векторной базе.
4. **Модуль Генерации:** Формирует промпт из текста жалобы и найденных законов. Локальная нейросеть (Ollama) пишет черновик ответа.
5. **Постпроцессинг:** Вставляет сгенерированный текст и ФИО заявителя в официальный бланк организации. Передает документ делопроизводителю на утверждение.

## 🛠 Стек технологий

**Инфраструктура и Backend:**

* **Язык:** Python 3.10+
* **Фреймворк:** FastAPI (REST API, Pydantic)
* **Брокер сообщений:** RabbitMQ
* **Базы данных:** PostgreSQL (метаданные), Qdrant / FAISS (векторы)

**Machine Learning & NLP:**

* **Оркестратор LLM:** LangChain
* **Локальный инференс LLM:** Ollama (квантованные модели GGUF)
* **ML фреймворк:** PyTorch
* **Энкодеры (Эмбеддинги):** BGE-m3, RuBERT
* **Извлечение сущностей (NER):** Natasha
* **OCR (Распознавание текста):** EasyOCR

---

## 🚀 Как происходит обработка обращения

1. Заявитель отправляет письмо на почту или заполняет форму на сайте.
2. Gateway получает текст (или скан) и отправляет его в брокер сообщений.
3. Система последовательно прогоняет текст через NLP-модуль, поиск НПА и языковую модель.
4. На выходе формируется готовый документ (Word/PDF).
5. Живой сотрудник читает черновик, нажимает кнопку «Утвердить», и письмо уходит заявителю.
