# Архитектура Системы Автоматической Генерации Ответов

Проект представляет собой сквозной ML-пайплайн для обработки входящих обращений граждан, семантического поиска нормативно-правовых актов (RAG) и генерации официальных ответов с использованием больших языковых моделей.

---

## 🏗 Общая схема конвейера обработки

```mermaid
graph LR
    %% Внешние системы
    Email([Почтовый сервер])
    Web([Веб-портал])
    API([API Клиент])
    Out([Каналы отправки])

    %% Базы данных
    DB_Meta[(Реляционная БД)]
    DB_Vector[(Векторная БД)]

    %% Модули системы
    M1[1. Препроцессинг и OCR]
    M2[2. Модуль NLP]
    M3[3. Модуль RAG]
    M4[4. Модуль Генерации]
    M5[5. Постпроцессинг]

    %% Интеграции
    Email -.->|Скан/Текст| M1
    Web -.->|Форма| M1
    API -.->|JSON| M1

    %% Пайплайн
    M1 ==>|Очищенный текст| M2
    M2 ==>|Текст| M3
    M3 ==>|Текст + Чанки НПА| M4
    M4 ==>|Черновик ответа| M5
    M5 -.->|Документ по ГОСТ| Out

    %% Работа с данными
    M2 -.->|Сохранение ФИО/Тематики| DB_Meta
    M5 -.->|Запрос реквизитов| DB_Meta
    M3 <.->|Поиск эмбеддингов| DB_Vector
    
    classDef default fill:#2b78e4,stroke:#1a52a3,stroke-width:1px,color:#fff;
    classDef db fill:#005b9f,stroke:#003d6b,stroke-width:1px,color:#fff;
    classDef ext fill:#11427b,stroke:#0a294d,stroke-width:1px,color:#fff;
    
    class DB_Meta,DB_Vector db;
    class Email,Web,API,Out ext;
